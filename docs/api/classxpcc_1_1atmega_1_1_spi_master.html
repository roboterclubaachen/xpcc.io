Detailed Description

The fast SPI clock speeds make it unreasonable to use an interrupt based approach to shifting out each byte of the data, since the interrupt handling might decrease performance over busy waiting especially for targets operating at low CPU frequencies. Therefore the asynchronous methods are implemented synchronously.

Warning
When the !SS pin is configured in input mode and pulled low, the hardware will switch into SPI slave mode. It is therefore necessary to configure the pin either in output mode or to pull it high.
