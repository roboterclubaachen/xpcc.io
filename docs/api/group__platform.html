#define XPCC_ISR_NAME (   vector)

On AVRs this maps to {vector}_vect. On Cortex-M this maps to {vector}_IRQHandler. On Hosted this maps to {vector}_isr.

Warning
These mappings are internal and can change without notice!
The CMSIS header files may define a macro with an identical name as an interrupt vector, such as XPCC_ISR_NAME(ADC) vs. the ADC peripheral. When defining interrupt vectors for compatibility, you MUST use this macro instead of manually defining it!
1 #define ADC ADC_CAN_TX // will not compile!
2 #define XPCC_ISR_NAME(ADC) XPCC_ISR_NAME(ADC_CAN_TX) // will compile
3 XPCC_ISR(ADC) {...}
Parameters
vectorThe name of the interrupt without any suffix (neither _vect, nor _IRQHandler).

#define XPCC_ISR (   vector,
  ... 
)

This macro allows the declaration of interrupt handlers using the name declared in the datasheet in C or C++ code on any platform.

On AVRs this maps to ISR(XPCC_ISR_NAME({vector}), args). On Cortex-M and Hosted this maps to void XPCC_ISR_NAME({vector})(void) args. extern "C" is automatically added in a C++ environment.

Parameters
vectorThe name of the interrupt without any suffix (neither _vect, nor _IRQHandler).
